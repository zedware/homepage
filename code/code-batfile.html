如何修复Windows中批处理文件的关联

    某年月日，安装了一个免费的编辑器，并将.BAT文件关联到了该程序。从而导致双击.BAT文件时不再自动执行，而是打开它进行编辑。点击QuickLaunch中批处理文件的快捷方式时，也是出现同样的效果。只好在该编辑器的配置界面中取消对.BAT文件的关联。然而双击.BAT文件或点击其快捷方式都会出现提示对话框“该文件没有程序与之关联来执行该操作，请在控制面板的文件夹选项中创建关联”。

    简单地Google一下，没有找到答案。猜测修复的办法是修改注册表KEY_CLASSES_ROOT\batfile。找了一台还正常的机器，参考其注册表，修改了：

[HKEY_CLASSES_ROOT\batfile\shell\open\command]
@="\"%1\" %*"

    问题解决了。
