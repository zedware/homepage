编码规范

　　使用编程语言需要遵循的最基本的规范自然是其语法规则，但是由于各种语言自身的特点，如果我们在编程时遵循某些特定的规则，就可以大大减少编程中出错的风险，提高代码的质量，减轻维护的代价。对于正式的项目，应该严格执行公司或项目组规定的编码规范，而且应组织代码检查和评审。有关软件过程管理的内容可以参看ISO 9000和SEI CMM/CMMI。
　　
　　对于不同的编程语言来说，具体的编码规范可以有很大的不同，但是其宗旨都是一致的，就是保证代码在高质量完成需求的同时具备良好的可读性、可维护性。例如我们可以规定某个项目的C语言程序要遵循这样的规定：变量的命名，头文件的书写和#include 等等。
